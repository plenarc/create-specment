## リリースノートフォーマット

### 変更の種類
- **追加** 新機能用
- **変更** 既存機能の変更用
- **非推奨** まもなく削除される機能用
- **削除** 現在削除された機能用
- **修正** バグ修正用
- **セキュリティ** 脆弱性修正用

### バージョン番号
このプロジェクトは[セマンティックバージョニング](https://semver.org/)に従います：
- **MAJOR** 非互換API変更用
- **MINOR** 後方互換機能追加用
- **PATCH** 後方互換バグ修正用

### リリースプロセス
1. package.jsonのバージョンを更新
2. 新バージョンでCHANGELOG.mdを更新
3. gitタグを作成: `git tag v1.0.0`
4. タグをプッシュ: `git push origin v1.0.0`
5. GitHub Actionsが自動的にnpmに公開
6. 変更ログノート付きでGitHubリリースを作成
